## Local providers

Run models entirely on your own machine — no API key, no data leaving your device.

| Provider | Notes |
| --- | --- |
| [Ollama](/providers/ollama) | Easiest local setup. Pull any model with `ollama pull`. |
| [LM Studio](/providers/lm-studio) | GUI app for running local models with an OpenAI-compatible API. |
| [vLLM](/providers/vllm) | High-performance inference server for GPU-accelerated local models. |

## Assigning models per feature

Go to **Settings → Model Selection** to choose which provider and model powers each Loophole feature:

- **Chat** — the main chat panel
- **Ctrl\+K** — inline quick-edit
- **Autocomplete** — inline code completions
- **Apply** — applies AI-suggested changes from chat to your files
- **SCM** — generates commit messages from your diff

<Tip>
  A common setup: use a large, capable model (like Claude Sonnet or GPT-4o) for Chat, and a small, fast model (like `qwen2.5-coder:1.5b` via Ollama) for Autocomplete.
</Tip>
