We build Loophole based on what users actually want. If you have an idea, please share it.

## How to suggest a feature

<Steps>
  <Step title="Search first">
    Check [existing issues](https://github.com/loophole-ai/loophole-ide/issues?q=is%3Aissue+label%3Aenhancement) to see if someone already asked. If yes, give it a thumbs up and add your thoughts in the comments.
  </Step>
  <Step title="Open a new issue">
    Go to [github.com/loophole-ai/loophole-ide/issues/new](https://github.com/loophole-ai/loophole-ide/issues/new) and pick **Feature request**.
  </Step>
  <Step title="Explain the problem">
    Start with the problem, not the solution. What are you trying to do? Why is it hard with Loophole today?

    For example: _"When I'm working on a big PR, I want to ask Loophole about my changes without copying the diff into chat. Right now there's no way to do that."_
  </Step>
  <Step title="Suggest a solution (if you have one)">
    Got an idea of how it could work? Share it. But it's totally fine to just describe the problem and let us figure out the how.
  </Step>
  <Step title="Mention alternatives">
    If you've tried workarounds or seen this feature in other tools, mention them. Helps us understand what you're going for.
  </Step>
</Steps>

## What makes a good feature request

- It solves a real problem you're hitting
- It would help more than just you (or you can explain why it's worth building anyway)
- It fits with what Loophole is trying to be — a fast, AI-native code editor

## What probably won't get built

- Big changes that turn Loophole into something else (a database tool, a design app, etc.)
- Features that only one person needs
- Things that would slow the editor down for everyone

We might still discuss them though. Doesn't hurt to ask.

## Vote on existing requests

Want to influence the roadmap? Go to [feature request issues](https://github.com/loophole-ai/loophole-ide/issues?q=is%3Aissue+label%3Aenhancement+sort%3Areactions-%2B1-desc) and give a 👍 to the ones you want. We look at this when deciding what to build next.
