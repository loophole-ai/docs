If something's broken in Loophole, please tell us. Even a tiny bug report helps.

## Before you file

Do a quick check:

1. Look at the [open issues](https://github.com/loophole-ai/loophole-ide/issues?q=is%3Aissue) to see if someone already reported it. If they did, give it a thumbs up and add any extra info in the comments.
2. Try restarting Loophole. (We know, we know.)
3. Check the [Troubleshooting page](/guides/troubleshooting) for common fixes.

Still broken? File an issue.

## How to file a good bug report

Head to [github.com/loophole-ai/loophole-ide/issues/new](https://github.com/loophole-ai/loophole-ide/issues/new) and pick **Bug report**.

A good bug report has:

### 1. A clear title

Bad: _"It doesn't work"_

Good: _"Chat panel goes blank after switching models"_

### 2. Steps to reproduce

List exactly what you did, like:

1. Open Loophole
2. Open the chat panel
3. Click the model dropdown
4. Select Claude
5. The chat panel goes blank

If we can't reproduce it, we can't fix it.

### 3. What you expected

_"I expected the chat panel to keep showing my conversation."_

### 4. What actually happened

_"The panel went completely white and stopped responding."_

### 5. Your setup

Go to **Help → About** in Loophole and copy the info there. Paste it in. It should include:

- Loophole version
- Operating system
- AI provider you were using

### 6. Screenshots or video (if it helps)

Drag images into the GitHub text box. For video, [Loom](https://www.loom.com/) is great.

### 7. Error messages

If you saw any error text, copy it in. You can also grab logs from **Help → Open Logs Folder**.

## After you file

A maintainer will usually respond within a few days. We might ask follow-up questions. The more you can help us reproduce it, the faster it gets fixed.

## Security issues

Found a security problem? **Please don't file it publicly.** Email us at [security@loophole.dev](mailto:security@loophole.dev) instead so we can fix it before bad actors learn about it.
