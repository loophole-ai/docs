---
title: "Working with Git"
description: "Commit, branch, and merge without leaving Loophole"
---

Loophole has Git built in, so you don't have to bounce between the editor and your terminal.

## See your changes

Click the Git icon on the left sidebar (it looks like a branch). You'll see:

- Files you've changed
- Files you've added
- Files you've deleted

Click any file to see a diff.

## Commit

<Steps>
  <Step title="Stage your changes">
    Hit the `+` next to a file to stage it. Or stage everything with the `+` at the top.
  </Step>
  <Step title="Write a message">
    Type your commit message in the box at the top.

    Not sure what to write? Click the **AI** button next to the box and Loophole will suggest one based on your changes.
  </Step>
  <Step title="Hit commit">
    Click the check mark or press `Cmd+Enter` / `Ctrl+Enter`.
  </Step>
</Steps>

## Branch and switch

Click the branch name in the bottom status bar to:

- Switch branches
- Make a new branch
- Check out a remote branch

## Push and pull

Click the sync icon in the bottom status bar. It pulls first, then pushes.

## Resolve merge conflicts

When you hit a conflict, Loophole highlights it in the editor. You'll see buttons above each conflict:

- **Accept current** — keep your version
- **Accept incoming** — take the other version
- **Accept both** — keep everything
- **Ask AI** — let Loophole figure out the right merge

The AI option works really well for tricky conflicts.
