Loophole lets you assign a separate AI model to each feature. This means you can use a powerful (and more expensive) model for complex agent tasks while using a fast, cheap model for autocomplete that fires on every keystroke.

## Features with independent model selection

| Feature | What it does | Recommended model tier |
| --- | --- | --- |
| **Chat** | Main AI chat panel and agent mode | Large / capable |
| **Ctrl\+K** | Inline quick-edit prompt | Medium / fast |
| **Autocomplete** | Inline code completions as you type | Small / fastest |
| **Apply** | Applies chat suggestions to your actual files | Medium |
| **SCM** | Generates commit messages from your staged diff | Small / medium |

## How to configure

<Steps>
  <Step title="Open Settings">
    Press `Ctrl+Shift+L` (`Cmd+Shift+L` on macOS) to open the Loophole settings pane.
  </Step>
  <Step title="Go to Model Selection">
    Click **Model Selection** in the settings sidebar.
  </Step>
  <Step title="Pick a provider and model for each feature">
    Each feature has its own dropdown. Select the provider first, then the model. Only providers you've configured with valid credentials will appear.
  </Step>
</Steps>

## Model overrides

For fine-grained control over how a model behaves, use **Model Overrides** (Settings → a specific model). Options include:

- **Max tokens** — cap the response length
- **Temperature** — adjust randomness (0 = deterministic, 1 = creative)
- **Reasoning** — enable/disable extended thinking for models that support it
- **Context window** — override the detected context window size
