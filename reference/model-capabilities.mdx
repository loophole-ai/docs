Not all models support all features. Loophole uses capability metadata to decide how to interact with each model. Here's a reference for the most commonly used models.

## Capability glossary

| Capability | What it means |
| --- | --- |
| **Reasoning** | Model performs chain-of-thought reasoning before answering |
| **Reasoning I/O** | Loophole can read the model's reasoning output (not just the final answer) |
| **FIM** | Fill-In-Middle: model is trained for autocomplete-style completions |
| **Vision** | Model can process images as input |
| **Tool use** | Model supports function/tool calling (required for agent mode) |

## Agent mode requirements

To use Agent mode, the selected Chat model must support **tool use**. If you pick a model that doesn't support tool calling, Loophole will fall back to Normal mode and warn you.
